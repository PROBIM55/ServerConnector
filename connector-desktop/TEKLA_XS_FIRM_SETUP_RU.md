# Настройка XS_FIRM для Tekla (единый путь)

## Цель

Обеспечить одинаковый путь `XS_FIRM` у всех пользователей.

Целевой путь:

- `C:\Company\TeklaFirm`

## Базовый принцип

- Путь `XS_FIRM` должен быть одинаковым у всех рабочих мест.
- Папка наполняется и обновляется через `Structura Connector`.
- Пользователь вручную папку не редактирует.

## Проверка на рабочем месте

1. Убедиться, что существует папка `C:\Company\TeklaFirm`.
2. Убедиться, что в коннекторе вкладка `Стандарт Tekla` показывает актуальный статус.
3. Проверить в конфигурации Tekla, что `XS_FIRM` указывает на `C:\Company\TeklaFirm`.

## Способ задания XS_FIRM

Рекомендуемый подход:

- централизованно задать `XS_FIRM` через корпоративный init/shortcut/конфигурацию запуска Tekla;
- не использовать пользовательские произвольные пути.

## Контроль после внедрения

- на пилотной группе проверить, что все запускают Tekla с одинаковым `XS_FIRM`;
- при расхождении путей исправить конфигурацию запуска.
