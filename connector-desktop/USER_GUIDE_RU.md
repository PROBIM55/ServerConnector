# Structura Connector — руководство пользователя

## Назначение

`Structura Connector` автоматически настраивает доступ к инфраструктуре по персональному токену.

Пользователь указывает только токен. Остальные параметры (идентификатор устройства, сессия, доступ к SMB) приложение получает с сервера.

## 1. Подготовка

Получите у администратора:

- персональный токен;
- ссылку на актуальный релиз.

Источник дистрибутива:

- `https://github.com/PROBIM55/ServerConnector/releases`

Файл установки:

- `Connector.Desktop.Setup.msi`

## 2. Установка и подключение

1. Установите `Connector.Desktop.Setup.msi`.
2. Запустите `Structura Connector`.
3. Введите токен в поле `Токен устройства`.
4. Нажмите `Подключиться по токену`.

После успешного подключения приложение автоматически:

- создает/обновляет сессию устройства;
- применяет сетевые правила доступа;
- подключает `\\62.113.36.107\BIM_Models`;
- включает фоновую проверку связи с сервером;
- сохраняет токен локально.

## 3. Обновление приложения

В блоке `Обновления` используется одна кнопка:

- `Проверить обновление` — проверка наличия новой версии;
- `Скачать и установить` — появляется, если обновление найдено.

Проверка выполняется:

- при запуске приложения;
- каждые 30 минут.

После обновления повторный ввод токена не требуется.

## 4. Использование токена на нескольких ПК

Один токен поддерживает одну активную сессию одновременно.

- При подключении на ПК-2 сессия на ПК-1 становится неактивной.
- Для возврата на ПК-1 нажмите `Подключиться по токену` на ПК-1.
- Для одновременной работы на двух ПК требуется отдельный токен для второй машины.

## 5. Проверка доступности сервера

Если подключение не выполняется, запустите в PowerShell:

```powershell
Test-NetConnection server.structura-most.ru -Port 443
Test-NetConnection server.structura-most.ru -Port 445
curl -UseBasicParsing https://server.structura-most.ru/health
```

Ожидаемый результат:

- `TcpTestSucceeded : True` для портов `443` и `445`;
- `StatusCode : 200` для `/health`.

## 6. Где хранить Revit-модели

Для работы через Revit Server центральные модели должны храниться в серверной папке проектов:

- `\\62.113.36.107\BIM_Models\Revit\Projects`

Важно:

- размещение центральных Revit-моделей в других папках не является штатным сценарием для Revit Server;
- служебный кэш Revit Server хранится отдельно и не используется как рабочая папка моделей.

## 7. Данные для обращения в поддержку

Передайте администратору:

- скриншот окна `Structura Connector`;
- последние строки из блока `Журнал (последние события)`;
- вывод команд:

```powershell
Test-NetConnection server.structura-most.ru -Port 443
Test-NetConnection server.structura-most.ru -Port 445
curl -UseBasicParsing https://server.structura-most.ru/health
(Invoke-WebRequest -UseBasicParsing https://api.ipify.org).Content.Trim()
```

## 8. Завершение работы

Закрытие окна сворачивает приложение в трей.

Для полного завершения:

1. Откройте трей Windows.
2. Нажмите правой кнопкой на иконку `Structura Connector`.
3. Выберите `Закрыть`.
